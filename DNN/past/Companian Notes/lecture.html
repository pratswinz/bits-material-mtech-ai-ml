Deep Neural Networks · Session 7

Convolutional Neural Networks

From raw pixels to learned features: how CNNs see the world. Small pattern detectors slide across an image. Layer by layer, a rich understanding builds up.

The big idea

Instead of connecting every neuron to every pixel, a CNN uses a small filter that slides across the image. The same filter finds the same pattern anywhere it appears.

What you will learn

Convolution, padding, stride, pooling, 1×1 convolutions, ReLU, softmax, FC layers, backprop for kernels, and the 5 design patterns of modern CNNs.

01 · Why this matters

Image Classification

We want a machine to look at a picture and say what is in it. This seems easy for a human, but it is surprisingly hard for a computer — until CNNs arrived.

What a computer actually sees. An image is a grid of numbers. A grayscale image stores one brightness value per cell (0 = black, 255 = white). A colour image stores three grids: one for Red, one for Green, one for Blue. A 224 × 224 colour image is \(224 \times 224 \times 3 = 150{,}528\) numbers.

The size problem

A naive approach connects every pixel to every neuron. A 224 × 224 colour image has 150,528 inputs. The first hidden layer alone would need tens of millions of weights. That is slow, uses too much memory, and ignores the structure of images.

Three key challenges

Viewpoint change

A cat photographed from the front looks very different from a cat photographed from the side. The pixel values change completely.

Translation invariance

A cat in the top-left corner sits at different pixel coordinates than a cat in the bottom-right. The classifier must recognise it either way.

Scale invariance

A small cat thumbnail and a large cat poster should both be classified correctly, even though the pixel values are different.

Key insight: Images have local structure — a pixel depends heavily on its neighbours, not on distant pixels. Nearby pixels form edges, corners, and textures. CNNs are designed to exploit this local structure.
02 · The core idea

What is a Convolutional Neural Network?

A CNN is a deep neural network that uses small, reusable filters instead of connecting every neuron to every input. This makes it efficient and naturally good at finding patterns wherever they appear.

Inspired by biology. Cells in the human visual cortex respond to oriented edges in small patches of the visual field. Deeper cells combine those to see corners and curves. Even deeper ones recognise faces and objects. A CNN does the same thing with numbers.

Three core principles

1. Local connectivity

Each neuron connects only to a small local patch of the input — its receptive field. It does not see the whole image at once.

2. Parameter sharing

The same filter is used at every position. One set of weights learns to detect, say, a vertical edge, and finds it everywhere in the image. This cuts parameters dramatically.

3. Equivariance

If the input shifts by two pixels, the feature map shifts by two pixels. The network's response moves with the object.

Key insight: CNNs replace "every neuron sees everything" with a better rule. Each neuron sees a small patch. The same filter is reused everywhere. This makes CNNs fast, compact, and naturally good at finding patterns wherever they appear.
03 · Hierarchical features

Why CNNs Work: Hierarchical Feature Learning

CNNs do not just detect edges. They build a hierarchy — each layer combines what the previous layer found into something more complex.

Analogy: assembling a picture from pieces. To recognise a face, first find edges. Then combine edges into eye shapes, nose outlines, and mouth curves. Then combine those parts into a face. A CNN does exactly this, one layer at a time.

Layer-by-layer feature hierarchy

Early layers (1–2)

Detect low-level features: edges at various angles, colour blobs, gradients. Kernels are small (3×3, 5×5). Receptive field is tiny.

Middle layers (3–4)

Combine edges into shapes and textures: corners, curves, stripes, eye shapes. Medium-level abstractions. Wider receptive field.

Deep layers (5+)

Detect object parts: wheels, faces, ears. Then combine those into complete objects. The receptive field now spans most of the image.

Key insight: Deep networks do not just see edges — they combine them into an increasingly abstract representation. Early layers look like Gabor filters; later layers look like object parts. No engineer designed these — they emerge from training on data.
04 · The core operation

The Convolution Operation

Convolution is the engine of a CNN. A small kernel (a grid of numbers) slides across the input, computing a dot product at each position. The result is a feature map.

Analogy: a template scan. The kernel is a tiny template: "this is what a vertical edge looks like." You slide it across the image. Where the template matches strongly, you get a big number. Where it does not match, you get a small or negative number. The output grid records how well the template matched at every position.

The formula

For an input image \(I\) and a \(k \times k\) kernel \(K\), the output at position \((i,j)\) is:

$$ O(i,j) \;=\; \sum_{m=0}^{k-1} \sum_{n=0}^{k-1} I(i+m,\; j+n) \cdot K(m,n) $$

The output size for an \(H \times W\) input, \(k\times k\) kernel, padding \(p\), and stride \(s\) is:

$$ H_{\text{out}} = \left\lfloor \frac{H - k + 2p}{s} \right\rfloor + 1 $$

Worked example: 6×6 input, 3×3 kernel, stride 1, no padding

Output size: \( \lfloor (6-3+0)/1 \rfloor + 1 = 4 \). The output is 4×4.

At the top-left: compute 9 products (kernel vs image patch) and sum. Then slide right by one pixel. Repeat 16 times to fill the 4×4 output.

Each entry in the output says: "how much does the kernel's pattern appear at this position?"

05 · Border control

Padding

Without padding, each convolution shrinks the feature map. In a deep network, this quickly reduces the map to nothing. Padding solves this by adding a border of zeros around the input.

Types of padding. Valid: no padding, output is smaller. Same: add zeros so output has the same spatial size as the input (for stride 1). Full: maximum padding — rarely used in CNNs.

Same padding calculation

For a \(k \times k\) kernel and stride 1, the padding needed to keep the same spatial size is:

$$ p = \frac{k - 1}{2} $$

For a 3×3 kernel: \(p = 1\). For a 5×5 kernel: \(p = 2\). Add \(p\) zeros on each side.

⚡ Change the padding

Slide to change padding (0 = valid, 1 = same for 3×3). Watch how the output size changes. With p = 1, the 6×6 input stays 6×6 after a 3×3 conv. With p = 0, it shrinks to 4×4.

0
Valid padding (p=0): 6×6 → 4×4
Key insight: Same padding lets deep networks preserve spatial resolution through many layers. Without it, feature maps shrink rapidly, losing spatial detail before the network has had a chance to learn complex patterns.
06 · Step size

Stride

Stride \(s\) controls how many pixels the kernel jumps at each step. Stride 1 = move one pixel at a time (default). Stride 2 = jump two pixels, halving the output size.

Why use stride > 1? It down-samples the feature map — the same role as pooling, but done inside the convolution itself. Stride-2 convolutions are used in many modern networks (e.g., ResNet) instead of separate pooling layers.

Output size with stride

$$ H_{\text{out}} = \left\lfloor \frac{H - k}{s} \right\rfloor + 1 \quad (\text{no padding}) $$

Example: \(H=6,\; k=3,\; s=2\): \(\lfloor(6-3)/2\rfloor + 1 = 2\). The output is 2×2.

⚡ Change the stride

Slide to change stride from 1 to 3. Watch the output shrink. With stride 1, every pixel is visited. With stride 2, every other pixel is visited — the output is half the size.

1
Stride 1: 6×6 → 4×4 (every position visited)
Trade-off: Higher stride reduces spatial resolution but cuts computation. You lose some detail, but the feature map is smaller and faster to process. Stride is a design choice — the network can learn to handle the lower resolution.
07 · Wider view, same weights

Dilated (Atrous) Convolution

A dilated convolution inserts spaces (holes) between kernel elements. It sees a larger region of the image with the same number of parameters.

Analogy: a spaced-out ruler. A 3-point ruler can span 3 units or 5 units — same 3 points, just spaced differently. The dilation rate controls the spacing. The ruler still has 3 weights; the covered span grows.

Effective receptive field

For kernel size \(k\) and dilation rate \(r\):

$$ k_{\text{eff}} = k + (k-1)(r-1) $$

Example: \(k=3, r=2 \Rightarrow k_{\text{eff}} = 3 + 2 = 5\). Same 9 weights see a 5×5 region.

Example: \(k=3, r=3 \Rightarrow k_{\text{eff}} = 3 + 4 = 7\). Same 9 weights see a 7×7 region.

⚡ Grow the receptive field with dilation

r=1: 9 weights, 3×3 effective field
Where it is used: Semantic segmentation tasks need large receptive fields without losing resolution. Stride or pooling both shrink the output, which loses pixel-level detail. Dilation keeps full resolution while widening the field of view. WaveNet uses dilated convolutions for long audio sequences.
08 · What comes out

Feature Maps and Multiple Channels

One filter produces one feature map. A CNN uses many filters in parallel, producing a stack of feature maps — one per filter. This stack is the 3D feature volume passed to the next layer.

Feature maps

A feature map is a 2D grid showing where in the image the filter's pattern was found. High values = strong match. Low or negative values = pattern absent. Different filters detect different patterns: horizontal edges, vertical edges, blobs, corners.

Multi-channel input (RGB)

A colour image has 3 channels (R, G, B). Each filter must also have 3 channels. For a 3×3 filter on a 3-channel input, the filter is a \(3 \times 3 \times 3 = 27\)-number volume. One dot product sums all 27 products to give one output number.

Parameter count

$$ \text{params} = (k \times k \times C_{\text{in}} + 1) \times C_{\text{out}} $$

where \(C_{\text{in}}\) = input channels, \(C_{\text{out}}\) = number of filters (= output channels). The spatial size of the input does not affect the parameter count.

⚡ Build the feature volume

Slide to add more filters. Each new filter adds one more feature map (one more depth slice). See how the output volume grows taller with more filters — but the parameter count grows linearly.

3 filters → feature volume 4×4×3. Params: (9×3+1)×3 = 84
Key insight: More filters = the network can detect more types of patterns. Modern networks use 64 → 128 → 256 → 512 channels as depth increases. Spatial size shrinks (via pooling), but channel count grows — maintaining information capacity.
09 · Field of view

The Receptive Field

The receptive field of a neuron is the region in the original input image that influences its value. Deeper neurons see larger regions — this is what lets deep networks understand large objects.

Analogy: Chinese whispers. Person 1 hears one phrase. By person 10, it has been shaped by many earlier people. A deep-layer neuron works the same way. Its value reflects a wide region of the original image, even though each step only looked at a small patch.

How it grows with depth

With 3×3 kernels and stride 1, each layer adds 2 to the receptive field:

  • Layer 1: 3×3
  • Layer 2: 5×5
  • Layer 3: 7×7
  • Layer \(n\): \((2n+1)\times(2n+1)\)

Stride-2 and pooling grow the receptive field faster.

⚡ Watch the receptive field grow

Layer 1: receptive field = 3×3
Key insight: Depth gives CNNs their wide "vision." Two stacked 3×3 conv layers cover the same receptive field as one 5×5 layer. Yet they use fewer parameters and add an extra nonlinearity. This is why modern networks prefer many small kernels over few large ones.
10 · Spatial compression

Pooling Operations

Pooling layers reduce the spatial size of feature maps. They have no learnable parameters — they apply a fixed rule (max or average) to small windows.

Analogy: taking the highlights. A pooling layer summarises a 2×2 patch into one number. Max pooling keeps the brightest activation. Average pooling takes the mean. You lose some detail, but gain position tolerance. A small shift in the image matters less after pooling.

Max pooling formula

$$ O(i,j) = \max_{\substack{0 \le m < k \\ 0 \le n < k}} I(i \cdot s + m,\; j \cdot s + n) $$

Worked example: 4×4 input, k=2, stride=2

Top-left 2×2 window: values 1, 3, 5, 6. Max = 6. Top-right: values 2, 4, 1, 2. Max = 4.

Bottom-left: values 7, 8, 2, 1. Max = 8. Bottom-right: values 3, 0, 4, 9. Max = 9.

Output: $$\begin{pmatrix}6&4\\8&9\end{pmatrix}$$ Size: \(\lfloor(4-2)/2\rfloor+1 = 2\). ✓

⚡ Max pooling vs Average pooling

Key insight: Pooling layers have zero learnable parameters. They only shrink the spatial size and add slight position-insensitivity. Max pooling is the most common; average pooling is used in Global Average Pooling (GAP) to replace FC layers.
11 · Channel mixer

1×1 Convolution

A 1×1 convolution seems trivial — a kernel of size 1 cannot look at neighbouring pixels. But it is one of the most powerful tricks in modern CNNs: it mixes information across channels without touching spatial structure.

Analogy: a recipe for mixing spices. Each spatial position has a rack of \(C_{\text{in}}\) "spices" (channel values). A 1×1 conv is a recipe: mix them in a certain ratio to produce \(C_{\text{out}}\) new blends. Apply 64 different recipes and you get 64 new channel values at every spatial position.

What it does

  • Channel reduction: 256 → 64 channels before a costly 3×3 conv. Cuts computation.
  • Channel expansion: 64 → 256 channels after a bottleneck.
  • Non-linear mixing: Combined with ReLU, it adds non-linearity across channels.

Bottleneck design: parameter savings

Standard: \(5\times5\) conv, 256 in, 256 out → \((25 \times 256 + 1) \times 256 \approx 1{,}638{,}400\) params.

Bottleneck (1×1 → 3×3 → 1×1): only \(16{,}448 + 36{,}928 + 16{,}640 \approx 70{,}000\) params — about 23× fewer.

⚡ Channel reduction with 1×1 conv

Slide the output channels from 1 to 32. See how a 1×1 conv changes the depth of the feature volume without touching the spatial size (H×W stays the same). The parameter count grows only with the number of output channels.

8
1×1 conv: 64 → 8 channels. Params: (1×1×64+1)×8 = 520
Key insight: The bottleneck design (1×1 → 3×3 → 1×1) is used in ResNet-50/101/152, Inception, and MobileNet. It achieves the same representational power at a fraction of the cost by reducing channels before the expensive spatial conv.
12 · The default activation

ReLU Activation

ReLU (Rectified Linear Unit) is the default activation function in CNNs. It is applied after every conv layer. It is simple, fast, and solves the vanishing gradient problem.

$$ \text{ReLU}(x) = \max(0,\, x) $$
Analogy: a light switch. If the signal is positive (the pattern was found), pass it through unchanged. If negative (pattern not found), output zero. Simple, and it keeps only the useful activations.

Why ReLU beats sigmoid/tanh in deep nets

Sigmoid problem

For large inputs, the sigmoid gradient approaches zero — this is the vanishing gradient problem. Gradients shrink as they pass through deep sigmoid layers; early layers barely learn.

ReLU solution

For positive inputs, the ReLU gradient is exactly 1 — no shrinkage. Gradients flow freely through ReLU layers. Networks can be trained 100+ layers deep.

⚡ Compare ReLU vs Sigmoid

Toggle between ReLU and sigmoid. Slide the input value \(x\). Watch the output and the gradient (slope of the curve at \(x\)). With sigmoid, the gradient shrinks near 0 for large |x|. With ReLU, it is always 0 or 1.

0.0
ReLU(0) = 0, gradient = 0
Watch out — Dying ReLU: If a neuron's input is always negative, its output is always 0. Its gradient is also always 0. It can never recover — it is "dead." To prevent this, use a small learning rate and careful weight initialisation. Or switch to Leaky ReLU (\(\text{max}(0.01x, x)\)).
13 · Converting scores to probabilities

Softmax Activation

The final layer of a classification CNN produces raw scores — one per class. Softmax converts these scores into a probability distribution: all outputs between 0 and 1, summing to 1.

$$ \text{softmax}(z_i) = \frac{e^{z_i}}{\sum_{j=1}^{C} e^{z_j}} $$
Why exponentiate? Exponentiation makes all values positive (we cannot have a negative probability). The division by the sum normalises them so they add to 1. Bigger raw scores get amplified more — the highest-score class dominates the probability.

Worked example: 3 classes

Raw scores: \(z = [2.0,\; 1.0,\; 0.1]\).

Exponentiate: \(e^{2.0}=7.389\), \(e^{1.0}=2.718\), \(e^{0.1}=1.105\).

Sum: \(Z = 7.389 + 2.718 + 1.105 = 11.212\).

Probabilities: \(p_1 = 7.389/11.212 = \mathbf{0.659}\), \(p_2 = 0.242\), \(p_3 = 0.099\).

Check: \(0.659 + 0.242 + 0.099 = 1.000\). ✓

14 · The decision head

Fully Connected Layer

After the convolutional layers extract spatial features, a Fully Connected (FC) layer collapses everything into a final class decision. Every neuron connects to every element of the input.

$$ \mathbf{y} = f(\mathbf{W}\mathbf{x} + \mathbf{b}) $$

where \(\mathbf{x} \in \mathbb{R}^n\) is the flattened feature vector, \(\mathbf{W} \in \mathbb{R}^{m \times n}\) is the weight matrix, and \(f\) is the activation (ReLU for hidden FC, Softmax for the output).

Flattening

Before the first FC layer, the 3D feature volume \((H \times W \times C)\) is reshaped into a 1D vector: \(H \times W \times C\) elements. For a \(5 \times 5 \times 16\) volume: \(5 \times 5 \times 16 = 400\) elements.

⚡ FC layer parameter count

Slide the number of neurons in the FC layer. Watch the parameter count grow. Compare with convolutions — FC layers dominate the parameter count and are the main reason for overfitting.

120
Input: 400 neurons. FC: 400→120. Params: 400×120+120 = 48,120
Watch out: FC layers need a fixed input size. If the image changes size, the flattened vector has a different length, and the weight matrix no longer fits. Convolutional layers are size-agnostic; FC layers are not. This is why Global Average Pooling (GAP) is often used instead.

16 · Counting carefully

Counting Parameters

Understanding how many parameters a layer has is essential for network design. The formula for conv layers is simple — and it does not depend on spatial size.

Conv layer

$$ \text{params}_{\text{conv}} = (k \times k \times C_{\text{in}} + 1) \times C_{\text{out}} $$

Pooling layer

Pooling layers have zero learnable parameters. They are fixed operations (max or average). Only hyperparameters: pool size \(k\) and stride \(s\).

17 · Architectural wisdom

Five Design Patterns

Modern CNN architectures are not arbitrary layer stacks. They use five proven design patterns that balance accuracy and computational cost.

1. Spatial Reduction + Channel Expansion

At each stage, halve the spatial size (via stride or pooling) and double the channels. This keeps the total "information volume" (\(H \times W \times C\)) roughly constant.

2. Repetitive Building Blocks

Use a fixed module (VGG block, ResNet block) and repeat it. Easier to scale and tune than an arbitrary layer stack.

3. Skip / Residual Connections

\(\mathbf{y} = F(\mathbf{x}) + \mathbf{x}\). The identity shortcut gives gradients a direct backward path, enabling 100+ layer networks (ResNet).

4. Bottleneck Designs

1×1 reduces channels → 3×3 conv on narrow volume → 1×1 restores channels. About 8× fewer parameters. Used in ResNet-50/101/152.

5. Multi-Scale Feature Extraction (Inception)

Apply 1×1, 3×3, and 5×5 convolutions in parallel, then concatenate. The network learns which scale is most useful for each feature. Used in GoogLeNet/Inception.

18 · How big should the network be?

Depth vs Width

Depth = number of layers. Width = number of channels per layer. Both matter. The question is how to balance them for a given task and budget.

Guidelines

When to go deeper

  • Large, diverse datasets (ImageNet, COCO)
  • Complex tasks needing hierarchical features
  • When you can use skip connections
  • Enough computational resources

When to go wider

  • Smaller, more uniform datasets
  • Need faster training/inference
  • Limited to shallow networks
  • Want ensemble-like diversity in filters

EfficientNet: compound scaling

EfficientNet scales all three dimensions together: depth \(d\), width \(w\), and resolution \(r\), subject to a fixed budget constraint:

$$ d^\alpha \cdot w^\beta \cdot r^\gamma = 2 $$
