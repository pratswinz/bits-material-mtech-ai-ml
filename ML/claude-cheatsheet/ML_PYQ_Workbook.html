TYPE 3 — Gradient Descent

Mid Mid EC-2
Core formula
\[\theta_j \leftarrow \theta_j - \alpha\frac{\partial J}{\partial\theta_j} \quad \text{Ridge: } +\frac{\lambda}{m}\theta_j\]

PYQ: Dec 2025 Q5 — \(x=[1,2,3], y=[2,5,7], \theta_0=0, \theta_1=1, \alpha=0.05, \lambda=2, m=3\)

Predictions \(h=[1,2,3]\), errors \(e=[-1,-3,-4]\)
\(\partial J/\partial\theta_0 = -8/3\), \(\partial J/\partial\theta_1 = -19/3 + 2/3\) (Ridge on \(\theta_1\))
Update: \(\theta_0=0.133, \theta_1=1.283\) · Cost drops 4.667 → 3.006 ✓
Negative gradient on \(\theta_j\) → increase that weight in GD update

TYPE 5 — Bias–Variance Tradeoff

Mid Mid EC-2
Core formula
\[R^2 = 1 - \frac{SS_{res}}{SS_{tot}} \quad \text{Train}\approx\text{Test high} \Rightarrow \text{bias}\]

PYQ: June 2026 Q5 — spam classifier learning curves

Model A: train 61%, val 60% → high bias (underfit); more data won't help
Model B: train 99%, val 67% → high variance (overfit); use L1/L2, fewer features
Improved: logistic + moderate features + L2 + cross-validation

TYPE 6 — Regularization (Ridge / Lasso)

Mid Mid EC-2
Core formula
\[J_{Ridge}=MSE+\lambda\sum\theta_j^2 \quad J_{Lasso}=MSE+\lambda\sum|\theta_j|\]

PYQ: 2024 Regular Q1 — overfitting with many features

(a) Many irrelevant features → Lasso (L1) zeros weights (feature selection)
(b) Large logistic weights → L2/L1 penalty shrinks them; tune \(\lambda\) via CV
Ridge update: \(\theta_j \leftarrow (1-\alpha\lambda/m)\theta_j - \alpha\partial MSE\) · Lasso subtracts \(\alpha\lambda\,\mathrm{sign}(\theta_j)\)
See also Ridge/Lasso numerical (2 iterations)

TYPE 7 — Decision Trees / Entropy / IG

Mid Mid EC-2
Core formula
\[H(S)=-\sum_k p_k\log_2 p_k \quad IG=H(S)-\sum_v\frac{|S_v|}{|S|}H(S_v)\]

PYQ: June 2026 Q2 — loan approval (10 applicants)

Parent: 5 Yes / 5 No → \(H(S)=1.0\) bit
Split Existing Loan: No branch 5Y/0N (H=0), Yes branch 0Y/5N (H=0) → IG = 1.0
Tree: Existing Loan=No → Approve; Yes → Deny (pure leaves)
Always pick attribute with maximum information gain at each node

TYPE 8 — Logistic Regression

Mid Mid EC-2
Core formula
\[\sigma(z)=\frac{1}{1+e^{-z}} \quad z=\theta^T x \quad \text{BCE: } -[y\log\hat p+(1-y)\log(1-\hat p)]\]

PYQ: June 2026 Q3 — min-max + logistic default prediction

Min-max credit scores → \(x'\in[0,1]\) · Sigmoid probabilities for each student
BCE loss \(L=0.864\) · One GD step: \(w_0^*=0.006, w_1^*=0.393, w_2^*=-0.424\)
Large positive coef ≠ guaranteed class (probability, not deterministic)
Decision boundary: set \(\theta^T x=0\) (or \(p=0.5\))

TYPE 9 — Feature Scaling

Mid Mid EC-2
Core formula
\[x'=\frac{x-x_{min}}{x_{max}-x_{min}} \quad x'=\frac{x-\mu}{\sigma}\]

PYQ: Practice T/F — scaling & trees

(a) GD needs scaling for fast convergence → TRUE
(b) Decision trees need scaling → FALSE (split-invariant)
(c) Min-max sensitive to outliers → TRUE
(d) Z-score maps to [0,1] → FALSE
Scale before GD/logistic; trees & Naïve Bayes often unaffected

TYPE 11 — Normal Equation / Dimensions

Mid Mid EC-2
Core formula

PYQ: 2023 Regular Q6 — \(m=28\) samples, \(n=4\) features + bias

\(\mathbf{X}\in\mathbb{R}^{28\times 5}\), \(\mathbf{y}\in\mathbb{R}^{28\times 1}\), \(\boldsymbol{\theta}\in\mathbb{R}^{5\times 1}\)
\(X^TX\in\mathbb{R}^{5\times 5}\) · wrong dimension on any axis → 0 marks
Add column of 1s for intercept before matrix multiply
