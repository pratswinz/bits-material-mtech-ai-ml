Classification Basics

Classification predicts a label. In binary classification, labels are usually 0 and 1. This module uses examples like job offered (1) vs not offered (0).

The intuition: Regression asks “how much?”. Classification asks “which class?”.
Everyday picture: An email filter puts each email in one tray: spam or not spam.

Why Linear Regression Fails for Classification

A least-squares line can produce values below 0 and above 1. These are invalid as probabilities.

The intuition: A probability model must stay in [0,1]. Linear output has no such bound.
Watch out: Outliers can move least-squares boundaries heavily for binary labels.

Sigmoid and Logistic Model

Logistic regression computes score \(z = w_0 + w^T x\), then applies sigmoid \(\sigma(z)=\frac{1}{1+e^{-z}}\).

\[\hat{y}=\sigma(z),\quad 0<\hat{y}<1\]
Worked example: For \(z=-0.8\), \(\hat{y}=\sigma(-0.8)=\frac{1}{1+e^{0.8}}\approx0.31\), so with threshold 0.5 we predict class 0.

Decision Boundaries

With threshold 0.5, prediction is 1 when \(z\ge0\). So decision boundary is \(w_0+w^Tx=0\).

The intuition: Linear score gives a straight boundary in feature space. Add polynomial features to create nonlinear boundaries in original coordinates.

Cross-Entropy Cost

For logistic regression we use cross-entropy:

\[J = -\frac{1}{m}\sum_i\left[y^{(i)}\log\hat{y}^{(i)} + (1-y^{(i)})\log(1-\hat{y}^{(i)})\right]\]
The intuition: Confident wrong predictions get very high penalty.

Regularization

Regularized objective adds penalty to control overfitting:

\[J_{reg}=J+\frac{\lambda}{2m}\sum_{j=1}^{d}w_j^2\]
Everyday picture: Fit the data, but keep the model simple — like packing a bag under a strict weight limit.

Confusion Matrix and Metrics

Confusion matrix gives TP, TN, FP, FN. Then compute accuracy, precision, recall, and F1.

\[Accuracy=\frac{TP+TN}{TP+TN+FP+FN},\quad Precision=\frac{TP}{TP+FP},\quad Recall=\frac{TP}{TP+FN}\]
Worked metrics: For TP=45, FN=5, FP=15, TN=135: Accuracy=0.90, Precision=0.75, Recall=0.90, F1=0.818.

ROC Curve and AUC

ROC plots TPR vs FPR as threshold moves. AUC summarizes ranking quality.

\[TPR=\frac{TP}{TP+FN},\quad FPR=\frac{FP}{FP+TN}\]
The intuition: Threshold is a slider. ROC shows all sensitivity-specificity trade-offs.

Class Imbalance

When positives are rare, accuracy can be misleading. A model predicting all negatives may still look good on accuracy.

Watch out: Optimize recall/precision trade-off for minority class, not raw accuracy.

Multiclass: One-vs-Rest and One-vs-One

Logistic regression is binary. For K classes: OvR uses K classifiers. OvO uses K(K-1)/2 classifiers.

StrategyClassifiersProsCons
OvRKSimple, fastMay be sensitive to imbalance in rest class
OvOK(K-1)/2Strong pairwise separationMore models to train

Summary and Exam Checklist

  • Logistic regression = linear score + sigmoid.
  • Use cross-entropy, not least squares, for classification loss.
  • Train with gradient descent.
  • Evaluate with confusion matrix metrics and ROC/AUC.
  • Handle class imbalance explicitly.
  • Extend to multiclass with OvR or OvO.
