What Is Regression?

Sometimes we want to predict a number, not a category. We want to know the price of a flat, a patient's blood pressure, or how long a battery lasts. That task is regression.

The intuition: Regression finds the rule that maps input features to a real number. Linear regression finds that rule as a weighted sum — a straight line, or a flat plane, or a hyperplane in higher dimensions.
Everyday picture: A property agent knows that larger flats cost more. She draws a line through past sales data. Given a new flat's area, she reads the price off the line. That is linear regression.

Formally, we have \(n\) training examples. Each example has input features \(\mathbf{x}^{(i)}\) and a real-valued target \(y^{(i)}\). We want a function \(h(\mathbf{x})\) so that \(h(\mathbf{x}^{(i)}) \approx y^{(i)}\).

Simple Linear Regression

Simple linear regression (SLR) uses one input feature \(x\) to predict \(y\). The model is a straight line:

\[ h(x) = w_0 + w_1 x. \]

\(w_0\) is the intercept — where the line crosses the \(y\)-axis. \(w_1\) is the slope — how much \(y\) changes per unit of \(x\). Both are weights (or parameters). We learn them from data.

The intuition: Picture a scatter plot of points. We want the line that sits "as close as possible" to all of them. Close means: the vertical gap between a point and the line (the residual) is small.
Everyday picture: Imagine pinning a rubber band flat across a scatter of pins on a board. The band balances the pull from all the pins. That is what the least-squares line does.

The residual

The residual for example \(i\) is the gap between prediction and truth:

\[ e^{(i)} = h(x^{(i)}) - y^{(i)}. \]

Positive residual → predicted too high. Negative → too low. We square each residual so positives and negatives do not cancel.

Worked example: residuals for three points
Line: \(h(x) = 1 + 2x\). Points: \((1,3),\,(2,5),\,(3,6)\).
  1. Predictions: \(h(1)=3,\quad h(2)=5,\quad h(3)=7\)
  2. Residuals: \(e_1 = 3-3 = 0,\quad e_2 = 5-5 = 0,\quad e_3 = 7-6 = 1\)
  3. Squared: \(0,\;0,\;1\). Sum \(= 1\).
Perfect for two points; off by one for the third.

The MSE Loss Function

We need a single score that tells us how bad our weights are. That score is the loss function. The most common choice is mean squared error (MSE):

\[ J(\mathbf{w}) = \frac{1}{n} \sum_{i=1}^{n} \bigl(h(\mathbf{x}^{(i)}) - y^{(i)}\bigr)^2. \]

We divide by \(n\) to get an average, so the loss does not grow just because we have more data.

The intuition: MSE is the "average squared miss distance". A loss of zero means perfect predictions. Training finds the weights that make \(J(\mathbf{w})\) as small as possible.
Everyday picture: Think of throwing darts at a target. MSE measures the average squared distance from the bullseye. The aim is to lower that number.
Worked example: computing MSE
Data: \((1,2),\,(2,4),\,(3,5)\). Model: \(h(x)=0.5+1.5x\).
  1. Predictions: \(h(1)=2.0,\quad h(2)=3.5,\quad h(3)=5.0\)
  2. Errors: \(e_1=0,\quad e_2=-0.5,\quad e_3=0\)
  3. Squared: \(0,\;0.25,\;0\)
  4. \(\text{MSE} = (0+0.25+0)/3 = \mathbf{0.083}\)

Lab: Loss Bowl

The contour plot shows MSE as a function of \(w_0\) and \(w_1\). Click anywhere on the plot to place your weights. The star is the minimum. Watch how your MSE changes.

Watch out: MSE penalises large errors heavily due to the square. One badly wrong prediction can dominate the loss. For datasets with outliers, try mean absolute error (MAE) instead.

Gradient Descent

Gradient descent (GD) is the most common way to find the best weights. The gradient of \(J\) at the current weights points uphill. We subtract a fraction of it to step downhill. We repeat until the loss stops improving.

The intuition: Imagine standing on a hilly surface in fog. You cannot see far, so you feel the slope under your feet and take one small step downhill. You keep stepping until the ground feels flat. That is gradient descent.
Everyday picture: The learning rate \(\alpha\) is how much you trust the slope reading under your feet. A rate of 0.01 means tiny, careful steps. A rate of 10 means giant leaps — you might jump over the valley.

The update rule for each weight is:

\[ w_j \leftarrow w_j - \frac{\alpha}{n} \sum_{i=1}^{n} \bigl(h(\mathbf{x}^{(i)}) - y^{(i)}\bigr)\, x_j^{(i)}. \]

For the intercept \(w_0\), set \(x_0^{(i)} = 1\) for every example.

Worked example: one GD step on the CHD dataset
3 patients: \(x_1\) = diastolic pressure, \(x_2\) = BMI, \(y\) = RR-CHD.
Initial: \(w_0=5,\; w_1=w_2=-0.03\). Learning rate \(\alpha=0.02\).
  1. Predictions: \(h_1=1.55,\quad h_2=1.85,\quad h_3=1.10\)
  2. Errors: \(e_1=-0.26,\quad e_2=+0.63,\quad e_3=-0.61\)
  3. Sums: \(\sum e_i = -0.24,\quad \sum e_i x_{1i} = -31.4,\quad \sum e_i x_{2i} = -11.65\)
  4. Updates: \(w_0 = 5 - \tfrac{0.02}{3}(-0.24) = \mathbf{5.0016}\),
    \(w_1 = -0.03 - \tfrac{0.02}{3}(-31.4) = \mathbf{0.1793}\),
    \(w_2 = -0.03 - \tfrac{0.02}{3}(-11.65) = \mathbf{0.0477}\)

Watch out: If \(\alpha\) is too large, the weights oscillate or diverge. If too small, training takes thousands of steps. Always plot the loss per iteration to check convergence.

Multiple Linear Regression

When we have more than one feature, we extend the line to a hyperplane:

\[ h(\mathbf{x}) = w_0 + w_1 x_1 + w_2 x_2 + \cdots + w_d x_d = \mathbf{w}^\top \tilde{\mathbf{x}}. \]

Here \(\tilde{\mathbf{x}} = (1, x_1, \ldots, x_d)\) prepends a~\(1\) for the intercept. The model is still linear in the weights, even if we use squared or log features.

The intuition: SLR is a line in 2-D. MLR with two features is a flat plane in 3-D. With ten features it is a hyperplane in 11-D. We cannot draw it, but the algebra is the same.
Everyday picture: A doctor predicts heart-disease risk from diastolic pressure and BMI. Each feature pulls the prediction up or down by its weight. That is multiple linear regression.

Lab: Feature Weight Sliders

Move the sliders to change each weight. See how the predicted output for a fixed input changes. MSE on three training points updates in real time.

Watch out: Adding more features always reduces training error. But it can raise test error if those features are just noise. Use cross-validation to pick the best set of features.

The Normal Equation

Gradient descent is iterative. There is also a closed-form solution — the normal equation:

\[ \hat{\mathbf{w}} = (\mathbf{X}^\top \mathbf{X})^{-1} \mathbf{X}^\top \mathbf{y}. \]

Here \(\mathbf{X}\) is the design matrix (each row is one example, each column is one feature plus a bias column of ones). \(\mathbf{y}\) is the target vector.

The intuition: The normal equation gives the exact answer in one shot — no iterations. It is like solving \(ax = b\) by writing \(x = b/a\) directly, instead of guessing and checking.
Everyday picture: Gradient descent is like finding the lowest point in a valley by walking downhill. The normal equation is like looking at a map and reading the coordinates of the lowest point directly.
Worked example: normal equation on the CHD data
Design matrix (3 rows × 3 cols including bias):
\[ \mathbf{X} = \begin{pmatrix} 1 & 80 & 35 \\ 1 & 80 & 25 \\ 1 & 100 & 30 \end{pmatrix}, \quad \mathbf{y} = \begin{pmatrix} 1.81 \\ 1.22 \\ 1.71 \end{pmatrix}. \]
Computing \((\mathbf{X}^\top\mathbf{X})^{-1}\mathbf{X}^\top\mathbf{y}\) gives the least-squares weights in one matrix operation.
Watch out: Inverting \(\mathbf{X}^\top\mathbf{X}\) costs \(O(d^3)\). With thousands of features it is slow. If \(\mathbf{X}^\top\mathbf{X}\) is singular (non-invertible), the normal equation fails. Use regularisation to fix this.

Model Evaluation

After training, measure how good the model is on held-out test data. Three standard metrics:

  • MAE — mean absolute error: \(\frac{1}{n}\sum|h(\mathbf{x}^{(i)}) - y^{(i)}|\). Easy to interpret: "we are off by this many units on average."
  • RMSE — root MSE: \(\sqrt{\frac{1}{n}\sum(h-y)^2}\). Same units as the target. Penalises large errors more than MAE.
  • \(R^2\) — coefficient of determination: how much of the target's variance does the model explain? \(R^2=1\) is perfect; \(R^2=0\) means no better than guessing the mean.
\[ R^2 = 1 - \frac{\sum(h(\mathbf{x}^{(i)})-y^{(i)})^2}{\sum(\bar{y}-y^{(i)})^2}. \]
Everyday picture: \(R^2 = 0.85\) means the model explains 85% of the spread in the data. The other 15% is noise it could not capture.

Lab: Metric Calculator

Drag the slope slider. Watch MAE, RMSE, and R² update live. Notice how RMSE punishes outlier predictions more than MAE does.

Watch out: Never pick your model or tune hyperparameters by looking at the test set. That leaks test information into the model and gives an overly optimistic score.

Bias, Variance & Overfitting

Every model has two sources of error. Bias is the error from wrong assumptions — for example, fitting a straight line to curved data. Variance is the error from being too sensitive to the training data.

The intuition: High bias = too simple → underfits (misses the pattern). High variance = too complex → overfits (memorises the training noise). The goal is to balance both.
Everyday picture: A biased ruler reads 2 cm short every time — always wrong in the same direction. A high-variance ruler gives a different answer each time — noisy and inconsistent. A good ruler is both accurate and repeatable.

Lab: Bias–Variance Tradeoff

Move the complexity slider. See how bias² (blue), variance (red), and total error (black dashed) change. Find the sweet spot where total error is smallest.

Watch out: Adding features always reduces training error. Test error first falls then rises. Use cross-validation to find the right complexity. Regularisation is the main tool to control variance.

Why Regularise?

A model with large weights fits training data closely, but is fragile. A tiny change in input causes a large change in prediction. Regularisation adds a penalty for large weights to the loss function.

\[ J_{\text{reg}}(\mathbf{w}) = \underbrace{\frac{1}{n}\sum_{i=1}^n (h(\mathbf{x}^{(i)})-y^{(i)})^2}_{\text{data fit}} + \underbrace{\lambda \cdot \Omega(\mathbf{w})}_{\text{penalty}}. \]

\(\lambda\) (lambda) is the regularisation constant. Large \(\lambda\) → strong penalty → simpler model. Small \(\lambda\) → trust the data more.

The intuition: Two forces fight each other. The data-fit term says "match the data". The penalty says "keep the weights small". \(\lambda\) is the dial between them.
Everyday picture: Think of packing a bag for a trip. The data says "pack everything you might need". The weight limit (regulariser) says "only the essentials". \(\lambda\) is how strict the weight limit is.

Lab: Lambda Dial

Drag the λ slider. Watch how the Ridge weights (bars) shrink as λ grows. At λ=0 the model overfits; at large λ all weights are near zero.

Ridge Regression (L2)

Ridge regression uses the sum of squared weights as the penalty:

\[ J_{\text{Ridge}} = \frac{1}{n}\sum(h-y)^2 + \lambda\sum_{j=1}^d w_j^2. \]

The gradient descent update becomes:

\[ w_j \leftarrow w_j(1 - \alpha\lambda) - \frac{\alpha}{n}\sum_{i=1}^n e^{(i)} x_j^{(i)}, \quad j \geq 1. \]

The factor \((1-\alpha\lambda)\) shrinks every weight by a fixed proportion at every step. Weights rarely reach exactly zero.

The intuition: Ridge pulls every weight toward zero with a spring. The stronger λ is, the stiffer the spring. All features stay in the model, but none dominates.
Worked example: Ridge, 2 iterations (CHD data)
Setup: \(w_0=5,\;w_1=w_2=-0.03\), \(\alpha=0.02\), \(\lambda=5\). Note \(\alpha\lambda=0.1\), shrinkage \(= 0.9\).
  1. Initial errors (same as GD): \(e_1=-0.26,\;e_2=+0.63,\;e_3=-0.61\)
  2. Iteration 1 sums: \(\sum e_i=-0.24,\;\sum e_i x_{1i}=-31.4,\;\sum e_i x_{2i}=-11.65\)
  3. After Iter 1: \(w_0=5.0016,\quad w_1=0.1823,\quad w_2=0.0507\)
  4. After Iter 2 predictions are much larger (weights grew fast because λ=5 is large relative to the data scale): \(w_0=4.59,\;w_1=-36.09,\;w_2=-12.40\)
In practice, scale features and tune λ via cross-validation to avoid divergence.
Watch out: Ridge shrinks weights but never sets them to zero. All features stay in the model. If you need sparse solutions (some weights exactly zero), use Lasso.

Lasso Regression (L1)

Lasso (Least Absolute Shrinkage and Selection Operator) uses the sum of absolute values as the penalty:

\[ J_{\text{Lasso}} = \frac{1}{n}\sum(h-y)^2 + \lambda\sum_{j=1}^d |w_j|. \]

The gradient of \(|w_j|\) is \(\text{sign}(w_j)\). The update rule is:

\[ w_j \leftarrow w_j - \frac{\alpha}{n}\sum_{i=1}^n e^{(i)} x_j^{(i)} - \alpha\lambda\,\text{sign}(w_j). \]
The intuition: Ridge pulls weights with a spring (force grows with weight size). Lasso nudges weights with a constant push in the direction of zero. A constant push can drive small weights all the way to zero — automatic feature selection.
Worked example: Lasso, 2 iterations (CHD data)
Same setup as Ridge. Since \(w_1=w_2=-0.03 < 0\): \(\text{sign}(w_1)=\text{sign}(w_2)=-1\).
  1. Iteration 1: \(w_0=5.0016,\quad w_1=0.2793,\quad w_2=0.1477\)
  2. Now both weights are positive, so \(\text{sign}(w_1)=\text{sign}(w_2)=+1\).
  3. After Iter 2: \(w_0=4.36,\quad w_1=-55.87,\quad w_2=-19.22\)
Both methods diverge at λ=5 on this small unscaled dataset. Always scale features first.
