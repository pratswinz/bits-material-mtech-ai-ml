Part I — Introduction to ML Q1 · Every Paper
Mitchell's Definition (memorise word-for-word):
"A computer program is said to learn from experience E with respect to some task T and some performance measure P, if its performance on T, as measured by P, improves with experience E."
T (Task) — what the system does (classify, predict, drive) P (Performance) — how success is measured (accuracy, MSE, F1) E (Experience) — the data it learns from (labelled dataset, reward signal)

Supervised Learning

Labelled data → learn f(x)→y

Classification: y = category (spam/ham, disease/healthy)

Regression: y = number (price, temperature)

Performance: Accuracy, F1, MSE, R²

Unsupervised Learning

No labels → find hidden structure

Clustering: K-means, group similar items

Density estimation: model P(x)

Performance: Silhouette score, inertia

Reinforcement Learning

Agent takes actions → receives rewards

Goal: maximise cumulative reward

No pre-labelled data — learns from feedback

E.g., self-driving car, game-playing AI

💡 Discriminative (e.g. Logistic Reg): models P(y|x) directly. Generative (e.g. Naïve Bayes): models P(x|y). Use Generative when you also need outlier/density estimation.
Part II — ML Workflow & Data Q2 · Every Paper

6 Data Quality Issue Types (memorise ALL)

  1. Missing values — blank/empty cell
  2. Outliers — implausibly extreme (Age=350, CGPA=87.5)
  3. Inconsistent format — same data, different style ("15-Aug-2022" vs "2022-08-15")
  4. Duplicate rows — identical records
  5. Unit mismatch / wrong data type — "120lbs" vs rest in kg; email "dev[at]mail.com"
  6. Inconsistent encoding — "Positive/Negative" mixed with "1/0" in same column

Missing Value Handling

Delete row — if <5% missing

Mean/Median impute — numerical, MAR

Mode impute — categorical

KNN impute — feature relationships exist

Outlier Detection

Z-score: \(z = \dfrac{x-\mu}{\sigma}\); flag |z|>3

IQR: outlier if \(x < Q_1 - 1.5\cdot IQR\) or \(x > Q_3 + 1.5\cdot IQR\)

IQR more robust to skewed data

Split StrategyWhen to useKey point

Stratified splitImbalanced classesEach split preserves class proportions ← exam favourite

Stratified K-foldSmall + imbalancedEvery fold keeps the class ratio

Feature Scaling

Min-Max: \(x' = \dfrac{x-x_{\min}}{x_{\max}-x_{\min}}\) → [0,1]

Z-score: \(x' = \dfrac{x-\mu}{\sigma}\) → mean=0, std=1

⚠️ Always scale before k-NN & Gradient Descent

Feature Engineering

One-hot encoding — categorical → binary

Binning — continuous → ranges

Polynomial features — add x², x·y

Interaction terms — x₁·x₂

⚠️ Feature Scaling T/F traps: Speeds GD = TRUE · Guarantees global min = FALSE · Prevents local optima = FALSE (linear reg is convex anyway) · Prevents XTX singularity = FALSE (singularity = linearly dependent features)
Part III — Linear Regression Q3 · Bias-Variance, Regularisation always tested

Linear Model

\[\hat{y} = \theta_0 + \theta_1 x_1 + \cdots + \theta_n x_n = \boldsymbol{\theta}^T\mathbf{x}\]

θⱼ = 2.5 → per unit ↑ in xⱼ, ŷ ↑ by 2.5

MSE Cost Function

\[J(\boldsymbol{\theta}) = \frac{1}{2m}\sum_{i=1}^m \bigl(\hat{y}^{(i)}-y^{(i)}\bigr)^2\]

Convex → guaranteed global minimum

GD Update Rule

\[\theta_j \leftarrow \theta_j - \alpha \frac{\partial J}{\partial \theta_j}\]

\[\frac{\partial J}{\partial \theta_j} = \frac{1}{m}\sum_i(\hat{y}^{(i)}-y^{(i)})x_j^{(i)}\]

Normal Equation

\[\boldsymbol{\theta} = (X^TX)^{-1}X^T\mathbf{y}\]

No iterations, no α. Cost: O(n³)

Use only for small n (≤10,000 features)

GD vs Normal Equation

Gradient DescentNormal Equation
Needs α (tune it)No α needed
Many iterationsOne computation
Use for large n, large mUse for small n
O(kn) per iterationO(n³) — infeasible for n=300k

Learning Rate α Traps

J increases over iterations → α too large → decrease α

J decreases very slowly → α too small → increase α

Try: 0.001 → 0.003 → 0.01 → 0.03 → 0.1 → 0.3 → 1

Polynomial Regression (degree d)

\[\hat{y} = \theta_0 + \theta_1 x + \theta_2 x^2 + \cdots + \theta_d x^d\]

Still linear in θ — same GD/normal-eq tools apply.

Part IV — Classification & Logistic Regression Q4

Why LR Fails for Classification (3 points = full marks)

  1. Output is unbounded (−∞ to +∞) — not a valid probability
  2. MSE + binary labels → non-convex cost — GD can get stuck in local minima
  3. Sensitive to outliers — one extreme point shifts the decision boundary

Fix: Use sigmoid to squash output + log-loss (cross-entropy) for convex cost.

Sigmoid Function

\[\sigma(z) = \frac{1}{1+e^{-z}}\]

z→+∞: σ→1  |  z→−∞: σ→0  |  z=0: σ=0.5

Decision boundary: θᵀx = 0

Log-Loss (Cross-Entropy)

\[J = -\frac{1}{m}\sum_i\bigl[y^{(i)}\log\hat{p}^{(i)} + (1-y^{(i)})\log(1-\hat{p}^{(i)})\bigr]\]

Convex → GD finds global min

Weight Interpretation (Log-Odds)

\[\log\frac{P(y=1)}{P(y=0)} = \boldsymbol{\theta}^T\mathbf{x}\]

θⱼ = change in log-odds per unit ↑ in xⱼ (others fixed)

θⱼ>0 → ↑xⱼ raises P(y=1)  |  θⱼ<0 → ↑xⱼ lowers P(y=1)  |  θⱼ≈0 → xⱼ has little effect

BehaviourWhat happens
J(θ)=0Perfectly fits training data → overfitting. Does NOT mean it works on unseen data.
Large weights (w=100)Sigmoid becomes a step function → hyper-confident → overfits. Fix: L2 regularization
θ=0σ(0)=0.5 for all x — predicts 50% probability everywhere, NOT J=0

One-vs-Rest (OvR)

Train k binary classifiers; predict class with highest confidence.

Softmax (Multinomial)

\(\text{softmax}(z_k)=\frac{e^{z_k}}{\sum_j e^{z_j}}\). Probs sum to 1.

Part V — Evaluation Metrics Q5 · Guaranteed Confusion Matrix Calculation

Confusion Matrix Layout

Pred POSITIVEPred NEGATIVE
Actual POSTP ✓ HitFN Missed (Type II)
Actual NEGFP False Alarm (Type I)TN ✓

Precision

\[\frac{TP}{TP+FP}\]

Minimise FP — spam filter, costly treatment

Recall (TPR / Sensitivity)

\[\frac{TP}{TP+FN}\]

Minimise FN — cancer, fraud detection

F1-score

\[\frac{2 \cdot \text{Prec} \cdot \text{Rec}}{\text{Prec}+\text{Rec}}\]

Harmonic mean — best for imbalanced data

Accuracy

\[\frac{TP+TN}{TP+FP+FN+TN}\]

⚠️ Misleading on imbalanced data!

Specificity (TNR)

\[\frac{TN}{TN+FP}\]

FPR (x-axis of ROC)

\[\frac{FP}{FP+TN} = 1-\text{Spec}\]

Precision vs Recall — When to use which?

  • Maximise Precision → FP is costly: spam filter, drug side-effects
  • Maximise Recall → FN is costly: cancer screening, fraud detection, COVID test
  • Use F1 → need a single balanced metric

ROC Curve & AUC

  • Plots TPR (y-axis) vs FPR (x-axis) as threshold varies 1→0
  • AUC=1.0: perfect  |  AUC=0.5: random (diagonal)
  • AUC >0.7 acceptable; >0.8 good; >0.9 excellent
  • ↓ Threshold → ↑ TPR, ↑ FPR (catch more positives, more false alarms)
🔴
Accuracy Paradox: If 95% of loans are rejected, a model that always predicts "reject" gets 95% accuracy but is useless — it never identifies approvals. Always check F1 and confusion matrix on imbalanced data.
Part VI — Decision Trees Q6 · Entropy + IG calculation guaranteed

Entropy Formula

\[H(S) = -\sum_{c=1}^{k} p_c \log_2 p_c\]

pᶜ = fraction of class c in set S

H=0: pure node (all same class)

H=1: 50/50 binary split (max impurity)

H=log₂k: k classes, all equal

Information Gain

\[IG(S,A) = H(S) - \sum_{v}\frac{|S_v|}{|S|}H(S_v)\]

= Parent entropy − weighted avg child entropy

Higher IG → better split → choose this attribute as root

Gain Ratio (C4.5)

\[\text{SplitInfo}(S,A)=-\sum_v\frac{|S_v|}{|S|}\log_2\frac{|S_v|}{|S|}\]

\[\text{GainRatio}=\frac{IG(S,A)}{\text{SplitInfo}(S,A)}\]

Penalises high-cardinality attrs (IDs, dates)

Entropy — Key Values

Pure [10,0]: H = 0 bits

50/50 [5,5]: H = 1.0 bit

[9,1]: H = −0.9log0.9 − 0.1log0.1 ≈ 0.469

[7,3]: H ≈ 0.881

[4 equal classes]: H = log₂4 = 2 bits

ID3 Algorithm — Steps

  1. Compute H(S) of the full dataset (parent)
  2. For each attribute A: compute entropy of each branch, then weighted average
  3. Compute IG(A) = H(S) − weighted avg entropy
  4. Select attribute with highest IG as root / next split
  5. Recurse on each branch; stop when node is pure or no attributes left
PruningHow it worksAdvantage
Pre-pruning (early stopping)Stop when: max depth, min samples, IG below thresholdFast; simple; but "myopic"
Post-pruningGrow full tree → prune branches that don't improve val accuracyCaptures complex patterns first, then simplifies
⚠️ Exam trap: "Assessing a DT using only training data is detrimental" = TRUE. A fully grown DT gets ~100% training accuracy but generalises poorly. Always evaluate on test/validation data.
💡 IG limitation: IG favours attributes with many unique values (IDs, dates) — use Gain Ratio to penalise high cardinality.

= −(0.667×−0.585) − (0.333×−1.585) = 0.918 bits

Step 2 — Entropy of each branch (Alcohol: 5 total):

H(Sob) = −(4/7)log₂(4/7) − (3/7)log₂(3/7) = 0.985

Step 3 — Weighted avg entropy:
(5/12)(0.722) + (7/12)(0.985) = 0.301 + 0.574 = 0.875

Step 4 — Information Gain:
IG(A) = 0.918 − 0.875 = 0.043 bits
