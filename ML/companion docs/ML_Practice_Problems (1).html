Part I — Introduction to Machine Learning

Q1–Q4 · Mitchell's T/P/E appears in every paper

Q1

ScenarioEasy A spam filter learns to classify emails as Spam or Not-Spam from a dataset of 50,000 manually labelled emails. After training, it is evaluated on its classification accuracy. In the context of Tom Mitchell's definition, identify T, P, and E, and state the type of learning.

T
Task: Classify incoming emails as Spam or Not-Spam (binary classification task)
P
Performance: Classification accuracy — the fraction of emails correctly classified. Alternative measures: Precision, Recall, F1-score (especially important for imbalanced classes).
E
Experience: The 50,000 manually labelled emails (features: subject line, sender, body text; label: Spam/Not-Spam).
✅
Type: Supervised Learning (labelled data) → Classification (predicting a category).
Q2

ScenarioEasy "A chess-playing program improves its performance at playing chess by playing thousands of games against itself and adjusting its strategy based on whether it wins or loses." Identify T, P, E and name the learning paradigm.

T
Task: Play chess (select optimal moves to win a game)
P
Performance: Percentage of games won (win rate against opponents of known strength)
E
Experience: Thousands of self-play games — each game outcome (win/loss) is the reward/feedback signal
✅
Type: Reinforcement Learning — the program receives a reward (win) or penalty (loss), not pre-labelled move data. No supervisor provides the correct move at each step.

Q4

ConceptualMedium A hospital wants to classify patient records as "disease present" or "disease absent" AND also flag records that are anomalous (unusual patterns not seen before). Should they choose a discriminative or generative classifier? Justify your answer.

✅
Generative classifier (e.g., Naïve Bayes).

Reason: A discriminative model (e.g., Logistic Regression) directly learns P(y|x) — it tells you the probability of disease given features. However, it cannot estimate how likely a particular input x itself is (P(x)).

A generative model learns P(x|y) — how data is generated for each class. From this, you can compute P(x) = ΣP(x|y)P(y) and detect anomalies: if P(x) is very low, the record doesn't fit any class's typical pattern → flag as anomalous.

Rule: If you need density estimation or outlier detection in addition to classification → choose Generative.

Q6

True/FalseMedium Mark each statement as TRUE or FALSE and explain why. (This exact question appeared in the 2024 Makeup paper.)

  1. Feature scaling ensures that gradient descent converges to the global minimum.
  2. Feature scaling speeds up gradient descent convergence.
  3. Without feature scaling, gradient descent may oscillate or take very long to converge.
  4. Feature scaling prevents the matrix XTX from being singular.
  5. Feature scaling prevents gradient descent from getting stuck in local optima for linear regression.
a
FALSE. Scaling does not change the topology of the cost surface. For linear regression, it's already convex (global min exists). For non-convex problems (neural nets), scaling doesn't eliminate local minima.
b
TRUE. Scaling makes the cost function contours more circular. Without scaling, elongated elliptical contours force many more iterations to reach the minimum. Scaling significantly reduces the number of steps needed.
c
TRUE. When feature scales differ greatly (e.g., income [0–100,000] vs age [0–100]), the gradient steps are dominated by the large-scale feature. The algorithm zigzags instead of moving directly towards the minimum.
d
FALSE. XTX becomes singular when features are linearly dependent (e.g., perfect multicollinearity), not because of scale differences. Scaling changes magnitudes, not linear independence.
e
FALSE. Linear regression's MSE cost function is convex — there are no local optima regardless of scaling. Scaling only affects convergence speed, not whether a local minimum exists.
Q7

ScenarioMedium You have a loan-approval dataset with 10,000 records: 8,700 Rejected, 1,300 Approved. A colleague randomly splits the data 70/30. You argue this creates a risk. Explain the risk and propose a better approach.

1
Risk of random split: With 13% approvals, a random 70/30 split may put most or all approved cases into training (or test) by chance. For example, if all 1,300 approvals end up in the training set, the test set has no positives — the model's test performance is meaningless. Even if some end up in both, the proportions will likely not match the true 87/13 ratio.
2
Better approach: Stratified Split. A stratified split ensures that each split (train and test) has the same class ratio as the full dataset — approximately 87% Rejected and 13% Approved. This preserves the real-world distribution in both splits.
✅
With stratified 70/30 split: Training = 7,000 records (≈6,090 Rejected, ≈910 Approved); Test = 3,000 records (≈2,610 Rejected, ≈390 Approved). The model is trained and evaluated on representative samples.
Q8

ConceptualMedium You train a model on an 800-sample dataset. Model A (single 80/20 split) achieves 91% accuracy. Model B (5-fold cross-validation) achieves 83% ± 4%. Which estimate is more trustworthy, and why?

1
Model B (5-fold CV) is more trustworthy. The single 80/20 split tests on only 160 samples — whether those 160 happen to be easy or hard examples is random. A lucky split gives inflated accuracy (91%).
2
5-fold CV tests on all 800 samples across 5 runs (each fold of 160 becomes the test set once). Every example contributes to the evaluation. The ±4% gives a confidence interval — much more stable.
✅
The 91% single-split estimate is likely inflated. The 83%±4% CV estimate better reflects real-world generalisation. Always prefer CV for small datasets.

Part III — Linear Regression

Q9–Q14 · Gradient descent, bias-variance, regularisation, and model comparison are all high-yield

Q9

ConceptualEasy You have a dataset with n = 500,000 features and m = 2,000,000 instances. Should you use Gradient Descent or the Normal Equation for linear regression? Justify with computational complexity.

✅
Use Gradient Descent.

The Normal Equation requires computing \((X^TX)^{-1}\), which involves inverting an n×n matrix (500,000 × 500,000). Matrix inversion is O(n³) — for n=500,000 this is \((5\times10^5)^3 = 1.25\times10^{17}\) operations. Completely infeasible even for modern hardware.

Gradient Descent scales to any n and m because it only computes the gradient (dot products), which is O(n·m) per iteration — manageable.

Additionally, with m=2,000,000, use Mini-batch GD (batch size 256) for GPU efficiency.
Q10

NumericalEasy You run gradient descent on a linear regression problem. After 50 iterations, the cost J(θ) is increasing instead of decreasing. What does this indicate and what should you do? Also, if J decreases very slowly over thousands of iterations, what does that suggest?

1
J increases → α (learning rate) is too large. The algorithm is overshooting the minimum: each step jumps past the bottom of the bowl and lands on the other side at a higher cost. Fix: decrease α (try dividing by 3: e.g., α=0.1 → α=0.03).
2
J decreases very slowly → α is too small. The steps are tiny, requiring thousands of iterations to converge. Fix: increase α (try multiplying by 3). Monitor that J doesn't start increasing.
✅
Recommended sweep: try α = 0.001, 0.003, 0.01, 0.03, 0.1, 0.3, 1 and plot J vs iterations. Pick the largest α where J consistently decreases.
Q11

ScenarioMedium You use Ridge Regression. Your model shows: Training RMSE = 8.4, Validation RMSE = 8.6 (both are high). Diagnose the problem and state what action to take with λ (increase or decrease). Explain the effect.

1
Diagnosis: High Bias (Underfitting). Training RMSE ≈ Validation RMSE and both are high → the model fails to capture the underlying pattern even on training data. The model is too simple or over-regularised.
2
Action: Decrease λ. High λ imposes a large penalty on coefficients → forces them towards zero → model becomes too simple (high bias). Reducing λ allows the model to fit the data more closely.
✅
Rule: ↑λ → ↑Bias, ↓Variance (simpler model). ↓λ → ↓Bias, ↑Variance. In underfitting, reduce λ. Also consider: adding more features, using polynomial features, or removing the regularisation entirely.
Q12

ConceptualMedium You have a dataset with 500 features, many of which are suspected to be irrelevant or redundant. Which regularisation method — Ridge (L2) or Lasso (L1) — is more appropriate? Explain what happens to the coefficients in each case.

✅
Lasso (L1) is more appropriate.

Lasso adds a penalty of \(\lambda\sum|\theta_j|\) (absolute values). Due to the geometry of the L1 ball, Lasso tends to drive coefficients of irrelevant features exactly to zero. This performs automatic feature selection — the result is a sparse model using only the most relevant features.

Ridge (L2) adds \(\lambda\sum\theta_j^2\) (squares). It shrinks all coefficients towards zero but never exactly to zero. All 500 features remain in the model (small but nonzero weights).

With 500 features and many suspected irrelevant ones, Lasso simplifies the model, improves interpretability, and reduces overfitting more effectively.
Q13

NumericalHard A linear regression model has current parameters θ₀=1.0, θ₁=2.0 and learning rate α=0.1. For a single training example (x₁=3, y=10), compute the prediction ŷ, the error, and the updated parameters after one gradient descent step.

1
Prediction:
ŷ = θ₀ + θ₁·x₁ = 1.0 + 2.0×3 = 7.0
2
Error: ŷ − y = 7.0 − 10 = −3.0
3
Gradients (m=1 example):
∂J/∂θ₀ = (ŷ−y)·x₀ = (−3.0)·1 = −3.0   (x₀=1 for the bias term)
∂J/∂θ₁ = (ŷ−y)·x₁ = (−3.0)·3 = −9.0
4
Updated parameters:
θ₀ ← 1.0 − 0.1×(−3.0) = 1.0 + 0.3 = 1.3
θ₁ ← 2.0 − 0.1×(−9.0) = 2.0 + 0.9 = 2.9
✅
The parameters moved towards better fitting the example (y=10). The new prediction would be: 1.3 + 2.9×3 = 10.0 (perfect fit for this single example — consistent with GD direction).
Q14

Part IV — Classification & Logistic Regression

Q15–Q19 · J(θ)=0 question appears in nearly every paper

Q15

ConceptualMedium Explain three reasons why linear regression should not be used directly for binary classification. What does logistic regression fix about each?

1
Unbounded output: Linear regression produces ŷ ∈ (−∞, +∞). For binary labels (0/1), we need a probability ∈ [0,1]. Values like −0.3 or 1.7 are meaningless.
Fix: Logistic regression uses the sigmoid function to squash the output to (0,1).
2
Non-convex cost function: Applying MSE loss with a sigmoid (or directly on binary labels) produces a non-convex cost surface with multiple local minima. Gradient descent can get stuck.
Fix: Logistic regression uses log-loss (cross-entropy), which gives a convex cost → GD guaranteed to reach global minimum.
3
Sensitivity to outliers: A single extreme example can shift the regression line significantly, moving the decision boundary in harmful ways.
Fix: The sigmoid's saturating nature makes logistic regression less sensitive to extreme outliers far from the decision boundary.
Q16

NumericalEasy A logistic regression model has parameters θ₀=−2, θ₁=1.5. For an input x₁=3, compute: (a) the linear score z, (b) the predicted probability P(y=1|x), and (c) the class prediction (threshold=0.5).

a
Linear score z:
z = θ₀ + θ₁·x₁ = −2 + 1.5×3 = −2 + 4.5 = 2.5
b
Predicted probability:
\(P(y=1|x) = \sigma(z) = \dfrac{1}{1+e^{-2.5}} = \dfrac{1}{1+0.082} \approx \dfrac{1}{1.082} \approx \mathbf{0.924}\)
c
Class prediction: P(y=1|x) = 0.924 ≥ 0.5 → Predict class 1 (Positive). The model is 92.4% confident this is a positive instance.
Q17

True/FalseMedium You train a logistic regression model and find that J(θ) = 0. Mark each statement TRUE or FALSE:

  1. The model will perform perfectly on any unseen test data.
  2. h_θ(x⁽ⁱ⁾) = y⁽ⁱ⁾ for every training example i.
  3. The model parameters θ₀ and θ₁ must both equal zero.
  4. J(θ) cannot reach exactly 0 for logistic regression.
  5. The model has overfit the training data.
a
FALSE. J=0 means perfect fit on training data → the model has memorised the training set. On unseen data it will almost certainly perform worse — this is overfitting.
b
TRUE. Log-loss = 0 only when every term is 0. Each term is 0 when h_θ(x⁽ⁱ⁾) = y⁽ⁱ⁾ exactly (probability = 1 for the correct class).
c
FALSE. If θ=0, then σ(0)=0.5 for all x — predicting 50% probability everywhere, which does NOT give J=0. Large weights (not zero) are needed to make σ(z) approach exactly 0 or 1.
d
FALSE. J can reach exactly 0 if the training data is linearly separable AND sufficiently large weights are used. The sigmoid approaches 0 or 1 asymptotically, but with very large |w|, it effectively reaches those values.
e
TRUE. J(θ)=0 → perfect training accuracy → the model has memorised the training data exactly → this is a classic sign of overfitting.
Q18

ConceptualMedium A logistic regression model for loan default prediction has coefficients: θ_income = −0.8, θ_debt = 1.2, θ_age = 0.05. Interpret each coefficient in terms of the log-odds of default.

1
θ_income = −0.8: For each unit increase in income (holding debt and age constant), the log-odds of default decrease by 0.8. Higher income → lower probability of defaulting. (Negative coefficient = protective factor.)
2
θ_debt = 1.2: For each unit increase in debt (others held constant), the log-odds of default increase by 1.2. Higher debt → higher probability of defaulting. This is the strongest predictor in the model. (Largest |θ|.)
3
θ_age = 0.05: For each one-year increase in age, the log-odds of default increase by 0.05. The effect is very small — age has minimal impact on default probability in this model.
✅
Log-odds = θ₀ + (−0.8)·income + 1.2·debt + 0.05·age. Convert to probability: P(default) = σ(log-odds). A large positive z → high default risk; negative z → low default risk.
Q19

NumericalMedium Plot (describe) the sigmoid function σ(wx) for w=1, w=10, w=100. How does increasing w lead to overfitting?

1
w=1: Smooth S-curve. σ transitions gradually from ~0 (x≪0) to ~1 (x≫0) over a wide range of x values. Uncertain near x=0 (σ=0.5).
2
w=10: Steeper S-curve. The transition from 0→1 happens much more rapidly near x=0. The model is more confident farther from the boundary.
3
w=100: Essentially a step function at x=0. For any x<0 (even x=−0.01), σ≈0. For any x>0 (even x=+0.01), σ≈1. The model is hyper-confident everywhere.
✅
Why overfitting: Very large weights create an extremely sharp decision boundary. The model perfectly separates training data (J→0) but even a tiny perturbation of a test example near the boundary causes extreme misclassification. It memorises training labels rather than learning generalisable patterns. Fix: L2 regularisation penalises large weights → keeps the sigmoid smooth → better generalisation.

Q20

NumericalMedium A cancer screening model is tested on 500 patients (50 have cancer, 450 do not). The confusion matrix is:

Predicted CancerPredicted No Cancer
Actual Cancer428
Actual No Cancer90360

Calculate Precision, Recall, Specificity, F1-score, and Accuracy for the Cancer class. Which metric is most important here and why?

1
Label TP, FP, FN, TN:
TP=42 (correctly predicted cancer), FN=8 (missed cancer cases), FP=90 (false alarms), TN=360 (correctly predicted no cancer)
2
Precision: \(\frac{42}{42+90} = \frac{42}{132} \approx 31.8\%\) — Only 1 in 3 predicted cancer cases is real.
3
Recall (Sensitivity): \(\frac{42}{42+8} = \frac{42}{50} = 84.0\%\) — Catches 84% of actual cancer patients. 8 patients were missed.
4
Specificity: \(\frac{360}{360+90} = \frac{360}{450} = 80.0\%\)
5
F1-score: \(\frac{2 \times 0.318 \times 0.840}{0.318+0.840} = \frac{0.534}{1.158} \approx 46.1\%\)
6
Accuracy: \(\frac{42+360}{500} = \frac{402}{500} = 80.4\%\)
✅
Recall is the most important metric here. Missing a cancer patient (FN=missed) means delayed treatment and potentially death. 90 false alarms (FP) result in unnecessary follow-up tests — inconvenient but not life-threatening. In medicine, maximise Recall for serious disease screening.
Q21

ScenarioEasy For each use case below, state whether you should maximise Precision or Recall, and explain why.

  1. Email spam filter (incorrectly flagging an important business email as spam is costly)
  2. COVID-19 test at an airport (missing an infected traveller could cause an outbreak)
  3. Fraud detection for credit cards (investigating every false alarm costs the bank heavily)
  4. Drug side-effect detection system (missing a dangerous side effect is unacceptable)
a
Maximise Precision. FP (flagging important email as spam) is costly — user misses important communication. A few spam emails in the inbox (FN) are acceptable. High precision = fewer false alarms.
b
Maximise Recall. FN (missing an infected traveller) could cause community spread. The cost of a false alarm (FP = unnecessary quarantine for a healthy traveller) is far less harmful. High recall = catch all infected people.
c
Context-dependent, but often Precision. If investigating each alert costs significant resources, many false alarms (FP) are expensive. Increase precision threshold so only high-confidence fraud triggers investigation. However, missing fraud (FN) also has costs — use F1 for balance.
d
Maximise Recall. Missing a dangerous side effect (FN) puts patients at risk. False alarms (FP = flagging a safe drug interaction) result in extra monitoring, which is manageable. Patient safety → maximise recall.
Q22

ConceptualEasy A model that predicts loan approval achieves 96% accuracy. The dataset has 96% rejections and 4% approvals. Is this model good? Explain the accuracy paradox and suggest a better metric.

1
No — the model is likely useless. A naive model that always predicts "Reject" would achieve 96% accuracy because 96% of examples are rejections. It never correctly identifies an approval.
2
Accuracy Paradox: On heavily imbalanced datasets, accuracy is dominated by the majority class. It rewards a model that ignores the minority class entirely.
✅
Better metrics:
• Precision for Approved class: of all predicted approvals, how many are real?
• Recall for Approved class: of all actual approvals, how many did we catch?
• F1-score: harmonic mean — balanced view
• AUC-ROC: threshold-independent measure
Check the confusion matrix to expose that the model predicts 0 approvals.
Q23

NumericalHard True labels = [1, 0, 1, 0, 1, 0], Predicted probabilities = [0.95, 0.80, 0.70, 0.40, 0.30, 0.10]. Compute TPR and FPR at threshold >0.75 and threshold >0.35. What does this tell us about the ROC curve?

1
At threshold >0.75: Predicted positive = [0.95, 0.80] → instances 1 and 2.
Instance 1: true=1 (TP); Instance 2: true=0 (FP)
Remaining: FN=2 (probs 0.70 and 0.30 with true=1), TN=2
TPR = TP/(TP+FN) = 1/3 ≈ 0.33; FPR = FP/(FP+TN) = 1/3 ≈ 0.33
ROC point: (0.33, 0.33)
2
At threshold >0.35: Predicted positive = [0.95, 0.80, 0.70, 0.40] → instances 1,2,3,4.
TPs: instances 1 (true=1), 3 (true=1) → TP=2; FPs: instances 2 (true=0), 4 (true=0) → FP=2; FN=1; TN=1
TPR = 2/3 ≈ 0.67; FPR = 2/3 ≈ 0.67
ROC point: (0.67, 0.67)
✅
Observation: Both points lie close to the diagonal (TPR ≈ FPR), suggesting this classifier performs near-randomly for these threshold values. As threshold decreases, TPR and FPR both increase. The closer the ROC curve bows towards the top-left corner, the better the classifier (higher AUC).

Part VI — Decision Trees

Q25–Q30 · Entropy + IG calculation is guaranteed 4–6 marks every year

Q25

NumericalEasy Compute the entropy for each node described below. Recall: \(H(S) = -\sum p_c \log_2 p_c\)

  1. Node with 10 instances — all class A (pure node)
  2. Node with 8 class-1, 8 class-2 instances (50-50)
  3. Node with 9 Yes, 5 No (14 total)
  4. Node with 4 equal classes (4 instances each, 16 total)
  5. Node with 14 class-1, 2 class-2 (16 total)
a
H = 0. p(A) = 10/10 = 1. H = −1·log₂(1) = −1·0 = 0 bits. Pure node — maximum information, no uncertainty.
b
H = 1.0 bit. p₁ = p₂ = 0.5. H = −0.5·log₂(0.5) − 0.5·log₂(0.5) = −0.5·(−1) − 0.5·(−1) = 0.5 + 0.5 = 1.0 bit. Maximum impurity for 2 classes.
c
H ≈ 0.940 bits. p(Yes)=9/14≈0.643, p(No)=5/14≈0.357.
H = −0.643·log₂(0.643) − 0.357·log₂(0.357) = −0.643·(−0.638) − 0.357·(−1.485) = 0.410 + 0.530 ≈ 0.940 bits
d
H = 2.0 bits. 4 equal classes → p_c = 1/4 each. H = −4·(1/4)·log₂(1/4) = −4·(1/4)·(−2) = 2 bits = log₂(4). Equal probabilities = maximum entropy for k classes.
e
H ≈ 0.544 bits. p₁=14/16=0.875, p₂=2/16=0.125.
H = −0.875·log₂(0.875) − 0.125·log₂(0.125) = −0.875·(−0.193) − 0.125·(−3) = 0.169 + 0.375 ≈ 0.544 bits. Low entropy — mostly pure.

Q28

ConceptualMedium A dataset has an attribute "Transaction-ID" that takes a unique value for every record. If you compute IG for Transaction-ID, it will be the highest of any attribute. Explain why this is a problem and how Gain Ratio addresses it.

1
Why Transaction-ID gives maximum IG: Every unique ID creates a separate leaf with exactly 1 example. Each leaf is perfectly pure (H=0). The weighted average entropy = 0 → IG = H(parent) − 0 = H(parent) = maximum possible. But each leaf holds only 1 training record — completely useless for generalisation.
2
The problem: ID3's IG metric systematically favours attributes with many distinct values — even when those attributes have no predictive value (like IDs, timestamps, names). This leads to overfitted, meaningless trees.
3
Gain Ratio (C4.5 fix):
SplitInfo measures the entropy of the attribute's own value distribution:
\(\text{SplitInfo}(S,A) = -\sum_v \frac{|S_v|}{|S|}\log_2\frac{|S_v|}{|S|}\)
For Transaction-ID with n unique values, SplitInfo = log₂(n) — very large.
GainRatio = IG / SplitInfo → large denominator → low GainRatio. The high-cardinality attribute is penalised.
✅
Gain Ratio normalises Information Gain by the attribute's own complexity. High-cardinality attributes (IDs, dates) get penalised; attributes that create meaningful, useful splits retain high GainRatio.
Q29

True/FalseMedium Mark each statement about decision tree overfitting as TRUE or FALSE:

  1. A fully grown decision tree achieves 100% accuracy on training data.
  2. Evaluating a decision tree only on training data is a reliable estimate of real performance.
  3. Post-pruning is generally more effective than pre-pruning.
  4. Pre-pruning can miss XOR-like interactions because it stops growth myopically.
  5. Adding more training data always increases decision tree training accuracy.
a
TRUE. A fully grown tree creates a unique path for each training example, perfectly memorising all labels. Training accuracy = ~100%, but test accuracy is typically much lower.
b
FALSE. Training accuracy is an optimistic (inflated) estimate — a fully grown tree achieves 100% on training data regardless of how good or bad the features are. Always evaluate on a held-out test/validation set.
c
TRUE. Post-pruning grows the full tree first (capturing all complex interactions) then prunes based on validation performance. Pre-pruning stops early and may prevent the tree from discovering patterns that only become clear deeper in the tree.
d
TRUE. XOR-like patterns (where no single feature predicts the outcome, but combinations of features do) require growing multiple levels before the interaction becomes apparent. Pre-pruning may stop before reaching this depth.
e
FALSE. More training data may actually decrease training accuracy for a decision tree. With more data, the tree must generalise more — it may no longer perfectly memorise every example with the same number of leaves. However, test accuracy typically improves.
Q30

NumericalHard Mixed Exam-Style Problem. A road accident dataset (12 records: 8 Major, 4 Minor) has two attributes. After computing IG: IG(Driver Condition) = 0.043, IG(Weather) = 0.010.
(a) Which attribute becomes the root node?
(b) The "Alcohol" branch has: {4 Major, 1 Minor}. Is this branch pure? If not, compute its entropy.
(c) You train a decision tree and get Training Accuracy = 98%, Test Accuracy = 62%. Diagnose and suggest two fixes.
(d) Your colleague suggests using Gain Ratio instead of IG. When is this especially important?

a
Root node: Driver Condition. IG(Driver Condition)=0.043 > IG(Weather)=0.010. Always select the attribute with the highest Information Gain as the root (ID3 rule).
b
Not pure — the branch has 2 classes (Major and Minor).
p(Major) = 4/5 = 0.8; p(Minor) = 1/5 = 0.2
H = −0.8·log₂(0.8) − 0.2·log₂(0.2) = −0.8·(−0.322) − 0.2·(−2.322) = 0.258 + 0.464 = 0.722 bits
Since it's not pure, ID3 would continue splitting this branch using the remaining attribute (Weather).
c
Diagnosis: High Variance (Overfitting). Training accuracy ~100% (the tree memorised training data) but test accuracy is only 62% — a huge 36% gap. The model fails to generalise.

Fix 1: Post-Pruning — grow the full tree, then prune branches where removing them improves validation accuracy.
Fix 2: Pre-Pruning — set max_depth (e.g., 4), min_samples_leaf (e.g., 5), or minimum IG threshold to prevent excessive growth.
Additional fix: Get more training data, or use an ensemble method (Random Forest).
d
Gain Ratio is especially important when:
1. Dataset has attributes with many unique values (IDs, names, timestamps)
2. Some attributes are continuous (after discretisation into many bins)
3. You want to avoid the bias towards high-cardinality attributes
Gain Ratio = IG / SplitInfo. High cardinality → high SplitInfo → low GainRatio → penalised. This is the key improvement in C4.5 over ID3.
